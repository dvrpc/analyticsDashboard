Reference

Definiton Use Case
Page Views

Page Views are recorded every time a page is loaded (or reloaded). Page Views are recorded when a user visits a page, refreshes a page or hits 'back' to return to a page. Page Views are an aggregate statistic.

If you are interested in seeing the total number of visits to a page, Page Views are a good starting point.

Sessions

Sessions are groups of user interactions that take place within a given time frame, for example 30 minutes. Sessions contain page views and other analytics. A session begins when a user opens a page and ends when the session time frame passes without any user interaction. Click here for more information.

Sessions are a good way to identify passive users vs active users. In general, more Page Views per Session means more active users.

Average Time on Page

Average Time on Page calculates how much time the average user spends on your page. Average Time on Page does not take into account Bounce Rates or Exits. This means the calculation is not affected by people who immediately leave or close out your web page.

You can use Average Time on Page to determine if people are actually using your page, or are just navigating to it and clicking away/exiting.

Subpaths

Subpaths are pages that exist within the scope of an existing URL. For example, dvrpc.org/resiliency/coastal is a subpath of dvrpc.org/resiliency. The main path is resiliency, and the subpath is coastal resiliency. Subpaths generally represent pages that are part of a set.

Knowing information about a pages Subpaths can help you determine if people are navigating further into a particular section.
